Neural Networks

Inspired by the human brain (loosely!)

A brain has ~86 billion neurons connected by synapses.

We mimic this with artificial neurons connected by weights.

How a Single Neuron Works

Input 1 × Weight 1
Input 2 × Weight 2
Input 3 × Weight 3
↓ ↓ ↓
SUM → Activation → Output

Like a tiny decision maker: "Based on these inputs and how important each one is... here's my answer!"

Layers = Levels of Understanding

Input Layer
Raw pixels
→
Hidden Layer 1
Edges & lines
→
Hidden Layer 2
Shapes & parts
→
Output Layer
"It's a cat!"

Each layer learns more abstract features.

Pixels → Edges → Eyes/Ears → Whole Cat! 🐱

"Deep" learning = many layers deep

How Does It Learn? (No Math, Promise!)

1
Forward Pass: Data flows through → makes a prediction
2
Check Error: Compare prediction vs actual answer → "How wrong was I?"
3
Backpropagation: Send error backwards → adjust weights
4
Repeat: Do this millions of times → get really good!

It's like a student taking exams, getting feedback, and improving each time! 📝

The smart notepad solution!

RNN vs LSTM — A Story

RNN: Like a student who can only remember the last few sentences of a lecture.

LSTM: Like a student who takes smart notes — writes down key points and reviews them when needed.

But both still read one word at a time... 🐌

What if you could read the entire page at once?

⚡ Transformers

"Attention Is All You Need" (2017)

The paper that changed everything

Key Insight: Don't read words one by one. Look at ALL words simultaneously and figure out which ones are related to each other.

This is called Self-Attention 🎯

Attention: The Restaurant Analogy 🍽️

Sentence: "The cat sat on the mat because it was tired"

Question: What does "it" refer to?

Attention mechanism: Looks at ALL words and scores their relevance:

• "cat" → HIGH attention score ✓

• "mat" → low attention

• "sat" → low attention

Like a spotlight that shines on the most relevant parts! 🔦

Why Transformers Won

⚡

Parallel Processing

Process ALL words at once, not one by one. Way faster!

🧠

Long Memory

Can connect word 1 to word 10,000. No forgetting!

📈

Scales Up

More data + more compute = better results. Always.

Transformers killed RNN/LSTM for most tasks overnight! 🪦

The Road to Modern LLMs

From counting words to understanding language

Let's trace the evolution step by step...

Bigram: The Simplest Language Model

Idea: Predict the next word based on just the previous word

Training text: "The cat sat on the mat"

Learns: "The" → "cat" (50%), "mat" (50%)

         "cat" → "sat" (100%)

         "sat" → "on" (100%)

Problem: "The" could be followed by ANYTHING. No context!

Trigram & N-grams: More Context

Bigram (n=2)

"___ sat" → ?

Only looks at 1 previous word

Trigram (n=3)

"___ cat sat" → ?

Looks at 2 previous words

The curse of N-grams:

More context = better predictions BUT exponentially more data needed!

For n=5 with 50,000 word vocabulary: 50,000⁵ = impossible combinations!

🎯 Next Token Prediction

The SECRET behind all LLMs

The entire magic of GPT, Claude, Gemini is just one simple task:

Given some text, predict the next word.

That's it. Really. The rest is just scale.

Next Token Prediction in Action

Prompt: "The capital of France is"

Model calculates probabilities:

"Paris" → 92% ✓

"Lyon" → 3%

"a" → 2%

"Berlin" → 0.1%

Picks: "Paris" → now predict next after "...France is Paris"

Autoregressive: Output feeds back as input, word by word! 🔄

Why Does Prediction = Intelligence?

1
To predict the next word well, you need to understand the text
2
To understand text, you need to learn facts, grammar, logic, reasoning
3
Train on the entire internet → learn almost everything humans have written
4
Scale to billions of parameters → emergent abilities appear!

"Compression of the internet into a neural network" — that's an LLM!

The Training Recipe 🍳

1
Pre-training: Read the entire internet. Learn language, facts, patterns. (Months on thousands of GPUs)
2
Fine-tuning (SFT): Show it conversations: "When asked X, respond like Y". Learn to be helpful.
3
RLHF/RL: Humans rate responses. Model learns which answers humans prefer.

Pre-training = knowledge

Fine-tuning = behavior

RL = alignment with human preferences

Chain of Thought (CoT)

Teaching AI to "think step by step"

Without CoT

Q: "If I have 3 apples and give away 1, then buy 5 more, how many?"

A: "8" ❌ (might get wrong)

vs

With CoT

Q: Same question + "Think step by step"

A: "Start: 3, give 1: 2, buy 5: 7" ✅

Reasoning Models

The evolution:

GPT-3: "Just answer" → often wrong on complex tasks

CoT Prompting: "Think step by step" → much better!

o1/o3/DeepSeek R1: Built-in reasoning → thinks before answering

Claude: Extended thinking → shows its work

Key insight: More "thinking" tokens = better answers

Just like humans — rushing through a math problem vs. working it out on paper 📝

SFT: Supervised Fine-Tuning

The "Show by Example" Method

You give the model conversation examples:

User: "What's the refund policy?"

Assistant: "Our refund policy allows returns within 30 days..."

Do this for thousands of examples → model learns the pattern!

Analogy: Like training a new employee by showing them how experienced people handle calls 📞

RL: Reinforcement Learning (RLHF)

The "Rate My Answer" Method

Step 1: Model generates 2+ different answers
↓
Step 2: Human says "Answer A is better than B" 👍👎
↓
Step 3: Train a "reward model" from these preferences
↓
Step 4: LLM learns to maximize the reward! 🏆

Analogy: Like a chef getting Yelp reviews and adjusting recipes! 👨‍🍳

SFT vs RL — Quick Comparison

SFT

"Do it exactly like this"

✅ Simple & reliable

✅ Full control over outputs

Like a recipe book

+

RLHF

"This answer is better"

✅ Captures human preferences

✅ Handles nuance better

Like taste testing

Modern LLMs use BOTH: SFT first, then RL to polish! 💎

🔥 Agentic AI

LLM in a Loop = Agent

The most exciting frontier in AI right now!

The Simple Insight

What if the LLM could:

1
Think about what to do
2
Act — call tools, write code, search the web
3
Observe the result
4
Loop back and decide what to do next?

This is the ReAct pattern: Reason + Act, in a loop! 🔄

The Coding Agent Example

Task: "Fix the bug in login.py"

Think: "I need to read the file first"

Act: read_file("login.py")

Observe: sees the code...

Think: "I see the bug — wrong variable name on line 42"

Act: edit_file("login.py", line=42, fix="...")

Observe: file updated

Think: "Let me verify by running tests"

Act: run_tests()

Observe: "All 15 tests passed ✅"

Think: "Bug fixed! Let me report back."

Generation + Verification = Agency

🧠 LLM GENERATES a plan/code/answer
↓
🔍 VERIFIER checks the output (compiler, tests, tools)
↓
❌ Error? → Feedback to LLM → Try again!
↓
✅ Success? → Deliver result!

The feedback loop is what makes it an agent, not just a chatbot!

Like a programmer with a compiler — write, compile, fix errors, repeat! 🔄

LangGraph

The Framework for Building Agents

Okay, so agents are cool. But how do we actually BUILD them?

Enter LangGraph — agents as graphs! 📊

Why LangGraph?

Simple agents: just a while loop + LLM call. Easy!

But real-world agents need:

  • Complex control flow — branching, parallel steps, conditional logic
  • State management — tracking what happened, what to do next
  • Human-in-the-loop — pause, ask user, resume
  • Persistence — save state, resume later
  • Streaming — show progress in real-time

LangGraph handles all this with an elegant graph-based model

LangGraph Core Concepts

🟢

Nodes

The "workers" — each node does one thing (call LLM, use tool, check result)

➡️

Edges

The "connections" — define flow between nodes, including conditions

📦

State

Shared memory — all nodes can read/write to it as data flows through

A LangGraph Agent — Visualized

📥 START: User Input
↓
🧠 LLM Node: Decide what to do
↓
🔀 Conditional Edge: Need a tool?
Yes → 🔧 Tool Node
No → 📤 Respond
↓ (tool result)
🧠 Back to LLM Node (LOOP!)

The graph keeps looping until the LLM decides it's done!

LangGraph Code — It's Simple!

# Define the graph
from langgraph.graph import StateGraph

graph = StateGraph(AgentState)

# Add nodes (the workers)
graph.add_node("agent", call_llm)
graph.add_node("tools", execute_tools)

# Add edges (the flow)
graph.add_edge(START, "agent")
graph.add_conditional_edges("agent", should_use_tool,
  {"yes": "tools", "no": END})
graph.add_edge("tools", "agent") # Loop back!

# Compile and run!
app = graph.compile()
